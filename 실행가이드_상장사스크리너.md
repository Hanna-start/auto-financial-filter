## 0. 지금까지 된 것 (요약)

가짜 데이터 대신, 옆 프로젝트 `dart-audit-extractor`의 **진짜 DART 재무**(`screener.db`)를
읽어오도록 연결을 끝냈다.

- 가격(1·4단계: 유동성·모멘텀) = yfinance
- 재무(2·3단계: 재무건전성·품질성장) = `screener.db` (실제, 연 단위)
- 정식 실행 파일: **`run_listed_screener.py`**
- 검증: 파일럿 실데이터로 결과가 손계산과 일치 + 전체 4단계(유동성→재무→품질→모멘텀) 정상 작동 확인
- yfinance 티커 버그 수정 완료: KOSDAQ도 `.KQ`로 정상 조회(전엔 `.KS`만 붙여 KOSDAQ이 누락됐었음)

⚠️ **지금 진짜로 분석되는 회사는 약 65개.** (2026-06-05에 100개 표본을 수집해 13→65로 늘림.
   100개 중 65개에 표준 재무가 있고 나머지는 스팩·리츠라 제외. 전체 목록은 3,965개)

## 1. VS Code에서 실행하기 (가장 먼저 할 것)

### ① 폴더 열기
VS Code → File → Open Folder → `D:\Agent_Project\auto-financial-filter` 선택.

### ② 터미널 열기
상단 메뉴 Terminal → New Terminal. (화면 아래에 입력창이 뜬다)

### ③ 필요한 패키지 설치 (처음 1번만)
터미널에 아래를 붙여넣고 Enter:

```powershell
py -m pip install -r requirements.txt
```

> yfinance·openpyxl 등 필요한 게 전부 여기 들어 있다. 한 번만 하면 됨.

### ④ 실행
```powershell
py run_listed_screener.py
```

성공하면 터미널에 단계별 통과 수와 최종 후보 종목이 뜨고,
폴더에 **`상장사_스크리너_결과_(날짜시간).xlsx`** 파일이 생긴다. 그게 결과다.

> 참고: 4단계(유동성→재무→품질→모멘텀) 전부 정상 작동 확인됨(65개사, 약 100초).
> 거래대금이 작은 회사는 1단계(유동성 50억 기준)에서 빠지는 게 정상이다.

## 2. 옵션: 기준 바꿔서 돌리기

기준을 바꾸고 싶으면 `run_listed_screener.py` 안의 `build_config()`에서 숫자만 고치면 된다:

| 항목 | 코드의 값 | 의미 |
|---|---|---|
| `min_trading_volume_krw` | 5_000_000_000 | 최소 거래대금(50억) |
| `max_debt_ratio_percent` | 200.0 | 부채비율 상한 |
| `min_revenue_growth_percent` | 0.0 | 최소 매출성장(YoY) |
| `min_operating_margin_percent` | 5.0 | 최소 영업이익률 |

가격 단계 없이 재무만 빠르게 보고 싶으면:
```powershell
py run_listed_screener.py --no-price
```

## 3. 다음 단계: 65개사 → 더 큰 표본 → 전체 상장사로 확대

지금은 `screener.db`에 약 65개사 재무만 있어서 65개만 분석된다.
더 보려면 **옆 프로젝트 `dart-audit-extractor`에서 재무 수집을 더 돌려야** 한다.
(그 프로젝트는 독립이라, 그쪽 폴더에서 실행한다. 이 폴더 코드는 안 건드림)

- 그 프로젝트의 수집기: `screener/collect.py` (DART API로 회사별 재무를 DB에 적재)
- 이미 한 것: `py screener/collect.py --pilot 100 --years 2022 2023 2024` (100개 표본 → 65개 적재)
- 더 늘리려면 `--pilot` 숫자를 키우면 된다(예: `--pilot 300`). 체크포인트가 있어 재실행해도 이미 받은 건 건너뛴다.
- DART API 키는 그 폴더의 `.env`에 이미 있음
- ⚠️ **1,000개 이상 = 일 한도(20,000콜)에 닿을 수 있음.** 그 전에 한도 감지·재개 처리(`screener/README.md`의 TODO)를
  붙여야 한다. **그러니 큰 범위로 가기 전에 범위·안전장치를 Claude와 함께 정하자.** (100개 표본은 무관)

> 즉, 확대는 "이 폴더"가 아니라 "dart-audit-extractor 폴더"에서 하는 별도 작업.
> 시작할 때 Claude에게 "전체 수집 도와줘"라고 하면 그 폴더 기준으로 단계별로 안내함.

## 4. 막히면 (간단 문제해결)

- **`py`를 모른다고 나옴** → `python run_listed_screener.py` 로 시도.
- **`screener.db를 찾을 수 없습니다`** → `dart-audit-extractor` 폴더 위치가 바뀌었는지 확인.
  바뀌었으면: `py run_listed_screener.py --db "새경로/screener.db"`
- **결과가 분석 대상(약 65개)보다 적거나 0개** → 정상일 수 있음(기준을 통과한 회사가 그만큼). 기준을 완화해 확인.
- **yfinance 관련 에러** → 인터넷 연결 확인. 그래도 안 되면 `--no-price`로 재무만 먼저 확인.
