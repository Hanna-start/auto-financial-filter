# 미국(S&P500·전체 거래소 상장) 선별 기준

> 실행 소스는 [`engine.py`]의 `US_CRITERIA` 프로파일 + `screen()`. 값을 바꾸려면 `engine.py`의 `US_CRITERIA`(+ 음수자본/피크 로직)를 수정한다. 국내 기준은 [기준_국내.md] 참고 — **한·미 독립(이원화)**. 국내는 원본 그대로 두고 미국만 시장 특성에 맞춰 조정했다.

적용 대상: `run_us_quarterly.py`(S&P500), `run_us_all_quarterly.py`(전체 거래소 상장).

## 3단계 게이트 (모두 AND)

| 단계 | 기준 | 값 | 국내와 차이 |
|---|---|---|---|
| ① 유동성 | 최근 20거래일 평균 달러거래대금 | **≥ $30M** | **다름**(국내 100억원 ≈ $7M; 미국 풀이 대형주 위주라 상향) |
| ② 재무건전 | 부채비율(총부채/총자본) | < 200% | 동일 |
| | 매출성장 (YoY) | ≥ 10% | 동일 |
| | TTM 영업현금흐름 | > 0 | 동일 |
| | **자본 게이트** | **자본총계 > 0 OR (자본총계 < 0 AND 이익잉여금 > 0)** | **다름**(아래) |
| ③ 품질성장 | TTM 영업이익률 | ≥ 10% | 동일 |
| | 이익 피크 | 충족 + **16분기 이상 이력** | **다름**(이력 가드 추가) |
| | 원가율 개선 | **미적용** | **다름**(US 제외) |

## 국내와 다른 4가지 (이원화 핵심)

1. **유동성 $30M** — 미국 풀은 대형주 위주라 이 게이트는 느슨하게 작동(정상). 실질 선별은 재무·품질 단계. 미국 확장 시 EDGAR 수집 전 `screen_us_liquidity.py`로 선적용해 수집 대상 축소.

2. **자본 게이트 = 음수자본 원인 판별** — 단순 `자본 > 0`은 자사주매입으로 자본이 음수가 된 우량주(예: 맥도날드 자본 −1.8B·이익잉여금 +70B)를 잘못 탈락시킨다. 그래서:
   - `자본 > 0` → 통과
   - `자본 < 0 AND 이익잉여금 > 0` → 통과 (자사주매입 결과 = 건강)
   - `자본 < 0 AND 이익잉여금 ≤ 0` → 탈락 (누적적자 = 진짜 자본잠식)
   - 효과: 부채비율 < 200%의 음수자본 허점도 동시에 차단.
   - ⚠️ 한계(수용): 스타벅스류(자사주·배당이 누적순이익 초과 → 이익잉여금까지 음수)는 우량이어도 탈락. 이익잉여금 부호가 "자사주 vs 적자"를 완벽히 가르진 못함.

3. **원가율 개선 제외** — 미국 풀은 소프트웨어 비중이 커 매출원가 구조상 무의미한 신호. 마진·이익 추세(영업이익률·이익피크)에 위임.

4. **이익피크 16분기 이력 가드** — EDGAR 이력 완결성이 종목마다 달라, 이력이 짧으면(신규상장 등) 거짓 피크로 잘못 통과할 수 있다. 16분기(4년) 미만 이력은 피크 판정에서 제외(탈락). `recent_quarters=20`(최근 5년)으로 절단한 뒤 적용.

> 그 외(부채비율<200%·매출성장≥10%·영업이익률≥10%·OCF>0)는 국내와 동일. 영업이익률은 GAAP상 SBC·상각 차감으로 미국에서 더 엄격히 작동하나 보수적 방향이라 유지.

## 표시용(필터 아님) — KR·US 공통

- PER·PBR·PSR·시총/TTM영업이익. **선별 미사용.**
- 분모(순이익·자본·매출)가 0에 가깝거나 결측이면 생기는 **비정상 배수(1,000배 초과)는 표시 제외('—')**. 정상 고배수(예: ARM PER 수백배)는 보존. (`engine.RATIO_MAX`)

## 데이터·주의

- 재무 = SEC EDGAR companyfacts(**us-gaap**) — 이익잉여금 = `RetainedEarningsAccumulatedDeficit`. 대형 ADR(TSM·ASML·Nokia 등)은 본국 IFRS로 20-F 제출 → us-gaap 빔 → **자연 제외**.
- 시세 = yfinance bulk + `market_cache` 폴백.
- 파이프라인: `build_us_all_universe.py` → `screen_us_liquidity.py`($10M 선필터) → `collect_us.py --index us_liquid_index.json` → `run_us_all_quarterly.py`.
